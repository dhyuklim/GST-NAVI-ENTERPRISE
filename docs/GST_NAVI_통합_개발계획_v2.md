## III. 개발 계획 — Phase별 상세

### 구분 기준

| Phase | 기간 | 핵심 원칙 |
|-------|------|----------|

### Phase 1: 대표이사 지시 반영 (2주)

> 목표: 회의 요구사항 직접 반영 + 다음 주 데모 가능

#### Week 1: 재무 + 생산 보강

| # | 작업 | 상세 | 공수 |
|---|------|------|:----:|
| 1-1 | **멘트/코멘트 입력 기능** | 각 대시보드 차트에 관리자 메모 추가/편집 (localStorage) | 2일 |
| 1-2 | **원가 구성비 표시 보강** | 현재 데이터로 가능한 범위 (매출원가/R&D/유지보수/판관비 비율) 시각화 개선 | 1일 |
| 1-3 | **생산 캐파/가동률 상세화** | 드릴다운 추가: 라인 클릭 → 월별 가동률 추이 + 비가동 사유 | 2일 |
| 1-4 | **1인당 생산성 월별 추이** | HR + 생산 연계: 부서별 1인당 생산성 6개월 추이 차트 | 1일 |

#### Week 2: SVC + 기간비교 고도화

| # | 작업 | 상세 | 공수 |
|---|------|------|:----:|
| 1-5 | **SVC 법인별 드릴다운** | 법인 클릭 → 장비 목록 + 워런티 상세 + 인력 배치 | 2일 |
| 1-6 | **기간 비교 시각화** | 비교 모드에서 나란히 바 차트 (이번달 vs 비교월) | 2일 |
| 1-7 | **데이터 완성도 표시** | 각 탭 상단에 데이터 소스 표시 (LIVE/MOCK/준비중) | 1일 |

**산출물**: 
- 재무: 판가 트렌드 + 기간 비교 + 원가 비율 시각화 + 멘트 기능
- 생산: 캐파 가동률 + 드릴다운 + 1인당 생산성
- SVC: 법인별 장비 + 워런티 + 드릴다운
- 전체: 데이터 완성도 라벨 (LIVE/MOCK)

| 2-4 | **생산 대시보드 실데이터 전환** | 엑셀 업로드 → 실 데이터 표시 + LIVE 라벨 | 2일 |

| 2-9 | **데이터 검증 리포트** | 업로드 후 품질 체크 (누락/이상값/중복) | 3일 |
